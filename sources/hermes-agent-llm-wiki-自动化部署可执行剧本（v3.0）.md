# 摘要
本文为适配火山引擎Coding Plan的Hermes Agent设计的v3.0版LLM Wiki自动化部署可执行剧本，将原抽象元编程工作流封装为可直接调用的Python脚本，大幅降低Agent执行门槛。剧本明确了分步执行、逐块校验的规则，覆盖环境预检、项目初始化、API配置全部署流程，Agent无需理解底层工作流概念即可完成部署。

# 核心观点
1. 【设计优化】将原抽象元编程工作流（如ingest、lint、建图等）全部封装为可执行Python脚本，Agent仅需按步骤调用，无需理解底层逻辑，大幅降低执行复杂度。
2. 【执行规则】严格要求按STEP顺序执行，每步执行后需校验输出是否符合预期结果，遇不符合项立即停止执行并上报错误，保障部署可靠性。
3. 【全链路校验】前置环境预检环节，对Python版本、pip可用性、Git安装、磁盘空间、火山引擎网络连通性5项阻塞项进行检查，未全部通过无法进入后续流程。
4. 【场景适配】专为macOS平台、火山引擎Coding Plan目标的Hermes Agent设计，内置火山引擎多节点连通性检查、API权限验证逻辑，适配目标部署环境。

# 详细内容
## 执行总规则
本剧本专为Hermes Agent设计，执行规则如下：
1. 按顺序执行每个STEP中的代码块，不得跳步执行
2. 每个代码块执行后，检查输出是否符合对应"预期结果"
3. 若输出不符合预期，立即停止执行并上报错误
4. 无需尝试理解或模拟ingest、lint、build graph等抽象概念，所有工作流已编码为可直接调用的Python脚本

## 分步部署流程
### STEP 0：环境预检
本阶段为阻塞项校验，检查部署所需的全部前置依赖，未通过无法进入后续流程。
#### 0.1 操作步骤
1. 在用户根目录创建`$HOME/check_env.py`文件，写入预检脚本（完整代码见原始资料），核心检查逻辑覆盖5项硬门槛：
   - Python版本 ≥ 3.10
   - pip3工具可用
   - Git已安装
   - 磁盘可用空间 ≥ 5GB
   - 火山引擎北京/上海节点网络连通
2. 执行预检命令：
```bash
cd ~
python3 check_env.py
```
#### 0.2 预期结果
- 控制台输出`[PASS] 全部通过，可继续部署`
- 自动生成预检报告文件`$HOME/.llm-wiki-check.json`
#### 0.3 异常处理
若执行失败，根据错误提示修复环境（如升级Python、清理磁盘、排查网络等）后重新执行预检。

### STEP 1：项目初始化
本阶段创建LLM Wiki的完整目录结构与基础配置文件。
#### 1.1 操作步骤
1. 在用户根目录创建`$HOME/init_project.py`文件，写入初始化脚本（完整代码见原始资料），核心功能为创建标准目录结构、生成环境配置模板、生成唯一部署标识。
2. 执行初始化命令：
```bash
cd ~
python3 init_project.py
```
#### 1.2 预期结果
- 自动创建`$HOME/llm-wiki-project/`完整目录结构，包含原始素材目录（论文、文章、书籍、会议记录等）、Wiki输出目录、知识图谱目录、脚本工具目录、日志与快照目录等
- 生成权限为600的`.env`环境配置模板文件，内置火山引擎默认配置项
- 生成唯一部署标识SETUP_ID，格式为`wiki-秒级时间戳-进程ID`

### STEP 2：火山引擎API配置
本阶段完成火山引擎API的配置与有效性验证。
#### 2.1 手动配置API Key
打开`$HOME/llm-wiki-project/.env`文件，将`VOLC_API_KEY=your_api_key_here`替换为从火山引擎控制台「API Key 管理」模块获取的实际API Key。
#### 2.2 API有效性验证
1. 在用户根目录创建`$HOME/verify_api.py`验证脚本（注意：原始资料中该脚本代码未完整提供，需补充完整API请求与校验逻辑后再使用），核心功能为加载`.env`配置、验证API Key的有效性与接口连通性。
2. 执行脚本完成验证后即可进入后续部署流程。

# 关键概念
## 可执行剧本（EXECUTABLE-PLAYBOOK）
本Wiki体系中专门为AI Agent设计的文档分类，文档内包含明确的可执行代码块、执行顺序规则、结果校验标准，无抽象工作流描述，Agent仅需按步骤执行即可完成预设目标任务。

## 阻塞项检查
部署前置校验阶段的硬性门槛检查项，任意一项未通过则无法进入后续部署流程，用于从源头规避部署失败风险，减少无效操作。

## SETUP_ID
LLM Wiki项目初始化时自动生成的唯一部署标识，格式为`wiki-秒级时间戳-进程ID`，用于关联本次部署全生命周期的所有配置、日志、快照资源，便于问题排查与版本回溯。
