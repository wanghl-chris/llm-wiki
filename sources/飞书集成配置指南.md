# 飞书集成配置指南

## 一、概述

Hermes Agent 已完成与飞书（Lark）的深度集成，支持：
- ✅ 通过飞书机器人与 Hermes 对话
- ✅ 定时任务自动推送结果到飞书
- ✅ 支持发送文本、图片、文件
- ✅ 支持群聊和单聊
- ✅ 自动重试机制（网络不稳定时）

---

## 二、功能说明

### 2.1 消息类型支持

| 类型 | 支持情况 | 说明 |
|------|----------|------|
| 纯文本 | ✅ 完整支持 | 发送普通文本消息 |
| Markdown | ✅ 支持 | 飞书原生 Markdown 语法 |
| 图片 | ✅ 支持 | 本地路径、URL 均可 |
| 文件 | ✅ 支持 | PDF、Word、Excel 等 |
| 富文本卡片 | ⚠️ 部分支持 | 需要使用飞书消息格式 |

### 2.2 图片发送说明

**已知问题**：
- 部分 JPG 图片使用上传图片 API 时会报错 `18033`
- **解决方案**：已修复为统一使用文件上传方式

**使用方式**：
- Hermes 会自动识别图片路径并转换为飞书文件发送
- 支持绝对路径、相对路径、`file://` 前缀
- Cron 任务中的 MEDIA 图片也会自动处理

## 四、飞书应用创建流程

### 步骤 1：创建企业自建应用
1. 打开飞书开放平台：https://open.feishu.cn/
2. 进入「开发者后台」→「创建应用」
3. 选择「企业自建应用」
4. 填写应用名称：`Hermes Agent 助手`
5. 上传应用头像
6. 点击「创建」

### 步骤 2：添加应用权限
在「权限管理」中添加以下权限：
- ✅ `im:message` - 发送消息
- ✅ `im:message:send_as_bot` - 以机器人身份发消息
- ✅ `im:file:upload` - 上传文件
- ✅ `contact:user.base:readonly` - 获取用户基本信息

**重要**：权限变更后需要企业管理员审批。

### 步骤 3：配置事件订阅
在「事件订阅」中：
1. 填写请求网址（需要公网可访问）
2. 配置 Encrypt Key 和 Verification Token
3. 添加事件：`im.message.receive_v1`（接收消息）
4. 保存并发布版本

### 步骤 4：获取凭证
在「凭证与基础信息」中获取：
- App ID
- App Secret

复制到 Hermes 配置文件中。

## 五、排障指南

### 问题 1：飞书收不到消息

**排查步骤**：
1. ✅ 检查 `config.yaml` 中 `feishu.enabled: true`
2. ✅ 检查 App ID 和 App Secret 是否正确
3. ✅ 检查 `home_chat_id` 是否正确（聊天 ID 不是群名称）
4. ✅ 检查应用是否已发布并添加到对应群聊
5. ✅ 检查网络是否能访问 `open.feishu.cn`

**获取正确的 Chat ID**：
- 在飞书中给机器人发一条消息
- 查看 Hermes 日志中的消息来源 `chat_id`
- 复制到配置文件的 `home_chat_id`

---

### 问题 2：图片发送失败（错误码 18033）

**现象**：
```
飞书 API 错误: 18033 - 图片上传失败
```

**原因**：
- 飞书图片上传 API 对部分 JPG 格式兼容性不好
- 图片文件过大（超过 10MB）

**解决方案（已实施）**：
- 统一使用文件上传 API（`/open-apis/im/v1/files`）
- 先上传为文件，再以文件消息发送
- 这个方法兼容性更好

---

### 问题 3：网络波动导致消息丢失

**现象**：
- Mac 刚唤醒时发送消息失败
- 网络切换时消息发送失败

**解决方案（已实施）**：
- 飞书适配器内置 3 次重试机制
- 重试间隔：15 秒 → 45 秒 → 120 秒
- Cron 任务执行前先做 DNS 健康检查

---

### 问题 4：机器人没有回复

**排查**：
1. 确认机器人在群聊中或已添加好友
2. 确认事件订阅配置正确
3. 确认 Hermes Gateway 服务正在运行
4. 查看 Gateway 日志有无报错

## 六、Gateway 服务管理

### 启动 Gateway
```bash
hermes gateway start
```

### 查看状态
```bash
hermes gateway status
```

### 查看日志
```bash
tail -f ~/.hermes/logs/gateway.log
```

### 重启 Gateway
```bash
hermes gateway restart
```

## 八、常见错误码速查

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| 10000 | 参数错误 | 检查请求参数格式 |
| 10013 | App ID 或 Secret 错误 | 核对配置文件 |
| 18033 | 图片上传失败 | 改用文件发送方式 |
| 9999166 | 权限不足 | 在开放平台申请对应权限 |
