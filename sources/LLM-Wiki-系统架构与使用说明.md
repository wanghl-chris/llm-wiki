# LLM Wiki 系统架构与使用说明

## 一、系统概述

LLM Wiki 是一个本地轻量化知识库系统，支持通过飞书交互进行知识积累、结构化整理和可视化展示。

### 核心特性
- ✅ 纯 Markdown 存储，无依赖，可随时用任何编辑器打开
- ✅ 交互式菜单操作，无需记忆命令
- ✅ 支持多种内容类型：文章、会议记录、网页剪藏、读书笔记
- ✅ 知识图谱可视化，自动发现内容关联
- ✅ 全文搜索功能
- ✅ Git 版本管理

## 二、系统架构

```
llm-wiki-project/
├── wiki_menu.py                    # 交互式菜单入口（17项功能）
├── llm-wiki-agent/
│   ├── wiki/                       # Wiki 输出目录
│   │   ├── index.md                # 知识索引首页
│   │   ├── sources/                # 知识页面（Markdown）
│   │   ├── entities/               # 实体抽取结果
│   │   ├── concepts/               # 概念抽取结果
│   │   └── syntheses/              # 内容综合结果
│   ├── raw/                        # 原始输入
│   │   └── inbox/                  # 收件箱（待处理文件）
│   ├── scripts/                    # 核心脚本
│   │   ├── create_wiki_page.py     # 处理文件生成 Wiki 页面
│   │   ├── quick_add.py            # 快速添加文本内容
│   │   ├── search_wiki.py          # 搜索知识库
│   │   ├── wiki_index.py           # 生成知识索引
│   │   ├── wiki_stats.py           # 知识库统计
│   │   ├── batch_import.py         # 批量导入文件夹
│   │   └── build_graph.py          # 构建知识图谱
│   └── graph/                      # 知识图谱输出
│       ├── graph.json              # 结构化图谱数据
│       └── graph.html              # ECharts 可视化页面
```

## 四、知识图谱说明

### 功能特点
- 基于页面的 `keywords` 字段自动构建关联
- 蓝色节点 = 知识页面
- 绿色节点 = 关键词
- 连线 = 页面与关键词的包含关系
- 关键词共现的页面之间也会建立关联

### 使用方法
1. 确保每个 Wiki 页面的 frontmatter 中有 `keywords` 字段
2. 执行菜单第 15 项「构建并打开知识图谱」
3. 在浏览器中查看交互式图谱：
   - 拖拽节点调整位置
   - 滚轮缩放
   - 悬停查看详细信息
   - 点击节点高亮关联节点

## 六、最佳实践

1. **及时填写 keywords**：添加内容时尽量填写关键词，这是知识图谱发挥作用的基础
2. **定期生成索引**：添加新页面后执行「重新生成知识索引」
3. **定期保存版本**：内容有更新时执行「Git 保存当前版本」
4. **善用图谱**：积累 10+ 页面后，知识图谱的价值会明显体现
5. **批量导入**：已有大量文档时，使用「批量导入文件夹」功能
